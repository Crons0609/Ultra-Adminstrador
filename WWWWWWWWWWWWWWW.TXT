# Corrección de gestión de cargos y permisos de empleados

Se ha identificado una limitación en el **Dashboard del Dueño** al momento de registrar empleados. Es necesario mejorar el sistema de cargos y permisos para que sea completamente flexible y adaptable a cualquier tipo de negocio.

## Problema actual

Al crear un empleado, únicamente aparecen los siguientes cargos predefinidos:

* Mesero
* Cocinero
* Caja
* Gerente

Esto limita el uso de la plataforma, ya que muchos negocios necesitan cargos diferentes según su operación.

## Comportamiento esperado

El sistema debe permitir dos opciones al crear un empleado:

### Cargos predefinidos

Mantener los cargos existentes:

* Gerente
* Mesero
* Cocinero
* Caja

### Cargo personalizado

Agregar una opción llamada **"Cargo personalizado"**.

Al seleccionarla, debe aparecer un campo de texto donde el Dueño pueda escribir cualquier cargo, por ejemplo:

* Bartender
* Recepcionista
* Supervisor
* Repartidor
* Cajero Principal
* Limpieza
* Seguridad
* Barista
* Encargado de Bodega
* Administrador
* Técnico
* Delivery
* Compras
* Marketing

O cualquier otro cargo que el negocio necesite.

El nombre ingresado debe almacenarse en Firebase y mostrarse en todo el sistema como el cargo oficial del empleado.

---

# Corrección del sistema de permisos

Actualmente existe un error importante.

Cuando se crea un empleado con un cargo personalizado, el sistema sigue mostrando mensajes o permisos relacionados con **Mesero**, por ejemplo:

> "El mesero no ha tomado la orden."

Esto es incorrecto.

## Comportamiento esperado

Los mensajes, funciones y permisos **no deben depender del nombre del cargo**, sino de los permisos asignados al usuario.

Por ejemplo:

* Si un empleado **no tiene el permiso "Tomar pedidos"**, el sistema no debe mostrar mensajes relacionados con un mesero.
* Si el empleado tiene un cargo personalizado como **Bartender** y no posee permisos para gestionar pedidos, el sistema no debe asumir que es un mesero.

Los textos y acciones deben basarse exclusivamente en los permisos habilitados para ese usuario.

---

# Sistema de permisos independiente

Los permisos deben funcionar de manera independiente del cargo.

Cada permiso debe poder activarse o desactivarse individualmente, por ejemplo:

* Tomar pedidos.
* Editar pedidos.
* Cancelar pedidos.
* Cobrar pedidos.
* Administrar caja.
* Ver reportes.
* Gestionar productos.
* Gestionar categorías.
* Gestionar inventario.
* Administrar empleados.
* Administrar mesas.
* Gestionar reservas.
* Ver estadísticas.
* Configurar impresoras.
* Recibir notificaciones.
* Administrar promociones.
* Gestionar clientes.

El cargo únicamente debe servir como una referencia organizativa. Los permisos serán los que definan qué acciones puede realizar cada empleado dentro del sistema.

---

# Objetivo

El sistema debe ser completamente flexible para adaptarse a cualquier tipo de negocio, permitiendo crear cargos personalizados y administrar los permisos de forma independiente. De esta manera, la plataforma no dependerá de cargos fijos ni mostrará mensajes incorrectos basados en un rol predefinido, sino que utilizará únicamente los permisos reales asignados a cada usuario.
