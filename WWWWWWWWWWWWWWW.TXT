# Correcciones, Mejoras y Nuevas Funcionalidades del Sistema SaaS

A continuación se detallan las mejoras, correcciones y nuevas funcionalidades que deben implementarse para garantizar un sistema estable, escalable y listo para producción.

---

# 1. Configuración de Zona Horaria

Actualmente el sistema no utiliza la hora correcta.

Es necesario configurar toda la plataforma para utilizar la **zona horaria oficial de Nicaragua**, de forma que todos los módulos trabajen con la misma referencia temporal.

La fecha y hora deberán reflejarse correctamente en:

* Inicio de sesión.
* Cierre de sesión.
* Caja.
* Apertura y cierre de turnos.
* Ventas.
* Compras.
* Inventario.
* Facturación.
* Reportes.
* Historial.
* Auditoría.
* Registros del sistema.
* Logs.
* Reservaciones.
* Pedidos.
* Notificaciones.
* Cualquier módulo que almacene fechas.

Toda la aplicación deberá utilizar una única zona horaria para evitar inconsistencias.

---

# 2. Implementar un Cron Job

Es necesario agregar un sistema de **Cron Job** o tarea programada que mantenga activa la aplicación y ejecute procesos automáticos.

Además de evitar que el servidor entre en estado de suspensión, deberá permitir ejecutar tareas como:

* Mantener activa la conexión con el servidor.
* Sincronizar información pendiente.
* Limpiar sesiones expiradas.
* Actualizar estadísticas.
* Generar respaldos automáticos.
* Verificar vencimiento de planes SaaS.
* Enviar recordatorios automáticos.
* Ejecutar tareas programadas por el sistema.
* Revisar notificaciones pendientes.
* Optimizar procesos internos.

---

# 3. Eliminar Empresas

Desde el panel del **Programador (Administrador General)** debe existir la posibilidad de administrar completamente todas las empresas registradas.

Agregar las siguientes funciones:

* Eliminar empresas.
* Desactivar empresas temporalmente.
* Reactivar empresas.
* Suspender empresas.
* Restaurar empresas eliminadas (papelera).
* Eliminar definitivamente una empresa y todos sus datos (con confirmación).
* Registrar en auditoría quién realizó la acción, fecha y motivo.

Antes de eliminar definitivamente una empresa deberá mostrarse un mensaje de confirmación para evitar eliminaciones accidentales.

---

# 4. Seguimiento GPS de Empleados

Agregar un módulo de geolocalización para los empleados que requieran trabajar fuera del establecimiento.

Este módulo permitirá visualizar en tiempo real la ubicación de:

* Repartidores.
* Conductores.
* Personal de entregas.
* Técnicos.
* Supervisores.
* Cualquier empleado autorizado.

Funciones del módulo:

* Ubicación en tiempo real.
* Historial de recorridos.
* Última ubicación registrada.
* Hora de actualización.
* Estado (Disponible, En ruta, Descansando, Desconectado).
* Distancia recorrida.
* Tiempo de trabajo.
* Tiempo detenido.
* Visualización sobre Google Maps u otro proveedor compatible.
* Filtros por empleado y fecha.

La ubicación deberá obtenerse únicamente con el consentimiento del usuario y respetando la configuración de permisos y la normativa de privacidad aplicable.

---

# 5. Corrección al Editar Empresas

Actualmente, al modificar la información de una empresa, los cambios no se reflejan correctamente.

Ejemplo:

Antes:

* Tipo de negocio: Comedor.

Después:

* Tipo de negocio: Entrega de alimentos.

El sistema continúa mostrando la información anterior.

Esto debe corregirse para que, una vez guardados los cambios:

* Firebase actualice los datos correctamente.
* La interfaz se refresque automáticamente.
* Se actualicen todas las vistas relacionadas.
* Los cambios permanezcan después de recargar la página.
* Se eliminen datos almacenados en caché cuando sea necesario.

---

# 6. Corrección del Panel de Planes SaaS

En el módulo **Planes SaaS**, los cambios realizados no permanecen guardados.

Debe corregirse para que cualquier modificación se almacene correctamente en Firebase.

Los cambios que deben persistir incluyen:

* Nombre del plan.
* Precio.
* Moneda.
* Duración.
* Descripción.
* Beneficios.
* Límite de usuarios.
* Límite de empleados.
* Espacio de almacenamiento.
* Cantidad de sucursales.
* Cantidad de productos.
* Estado del plan.
* Color del plan.
* Icono.
* Orden de visualización.
* Funciones habilitadas.

Cada modificación deberá reflejarse inmediatamente en la interfaz y mantenerse después de actualizar la página.

---

# 7. Integración con Lectores de Código de Barras y QR

El sistema deberá ser compatible con dispositivos físicos de lectura de códigos.

Debe permitir conectar:

* Lectores USB.
* Lectores Bluetooth.
* Lectores inalámbricos.
* Lectores de Código de Barras (EAN, UPC, Code 128, etc.).
* Lectores de Código QR.

Cuando el cursor esté sobre el campo correspondiente, el lector deberá completar automáticamente el código sin necesidad de escribirlo manualmente.

Funciones adicionales:

* Registrar productos mediante escaneo.
* Buscar productos por código.
* Actualizar inventario mediante escaneo.
* Agregar productos a una venta.
* Agregar productos a una compra.
* Registrar activos.
* Registrar equipos.
* Registrar vehículos.
* Registrar herramientas.
* Registrar insumos.

El código leído deberá guardarse permanentemente en Firebase para que pueda utilizarse en búsquedas futuras y acelerar la gestión del inventario.

---

# 8. Corrección de Configuración de la Página Pública

Actualmente, los cambios realizados en la personalización de la página pública no permanecen guardados.

Debe corregirse para que toda la configuración se almacene correctamente y se recupere cada vez que se abra el panel.

Esto incluye:

* Colores principales.
* Colores secundarios.
* Fondo.
* Logotipo.
* Banner.
* Imágenes.
* Galerías.
* Videos.
* Tipografía.
* Iconos.
* Redes sociales.
* Información de contacto.
* Horarios.
* Tema claro/oscuro.
* Botones.
* Diseño.
* Animaciones.
* Configuración SEO.
* Favicon.

Cada cambio deberá:

* Guardarse inmediatamente en Firebase.
* Reflejarse en la vista previa.
* Permanecer después de cerrar sesión.
* Mantenerse después de actualizar la página.
* Aplicarse automáticamente en la página pública del negocio.

---

# 9. Mejoras Generales de Persistencia

Es necesario revisar toda la aplicación para asegurar que cualquier dato editado o creado se comporte de forma consistente.

Todas las operaciones de:

* Crear.
* Editar.
* Eliminar.
* Activar.
* Desactivar.
* Cambiar configuraciones.
* Subir imágenes.
* Cambiar colores.
* Modificar permisos.
* Actualizar productos.
* Actualizar empresas.
* Modificar planes SaaS.

deberán:

* Guardarse correctamente en Firebase Realtime Database.
* Reflejarse inmediatamente en la interfaz sin necesidad de recargar la página.
* Sincronizarse en tiempo real con todos los usuarios autorizados.
* Mantener la información después de cerrar sesión.
* Mostrar mensajes claros de éxito o error.
* Registrar la acción en el historial y auditoría del sistema cuando corresponda.

El objetivo es que toda la plataforma funcione de forma estable, fluida y confiable, evitando pérdidas de información y garantizando que cualquier cambio realizado por el Programador, Dueño o Gerente se almacene y visualice correctamente en tiempo real.
